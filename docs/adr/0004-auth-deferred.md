# ADR-0004 — 인증 유보, 쓰기 규칙 개방

## 상태

완료 — 치지직 OAuth 2.0 + jose JWT로 구현 (ADR-0008 참조)

## 결정

현재 인증을 구현하지 않는다. Firestore 규칙은 `allow read, write: if true`로 두어 누구나 읽기·쓰기·삭제할 수 있다. 관리자 로그인은 **나중에 추가**하며, 그때 쓰기 작업을 관리자에게로 제한한다.

## 이유

- 연 2–3주만 활발히 운영되는 비공식 취미 도구이며, 데이터 손상 시 손실이 낮다.
- 초기엔 소수의 신뢰된 운영자만 결과를 입력하므로 실질 위험이 작다.
- 인증을 먼저 세우는 비용 대비 초기 효용이 낮아, 핵심 기능(기록·티어·전적)을 우선한다.

## 결과

- **현재의 신뢰 모델은 "악의적 사용자가 사이트를 발견하지 않는다"에 의존한다.** 공개 배포 상태에서 누구나 경기/스트리머를 추가·삭제할 수 있다.
- 중복 방지(소프트 가드)와 "관리자 입력" 전제(ADR 없음, 도메인 관례)는 **관리자 로그인이 들어와야 비로소 강제된다.** 그 전까지는 관례일 뿐이다.
- 관리자 로그인 추가 시: Firestore 규칙을 인증 기반으로 전환하고, 쓰기 경로(경기/스트리머 추가·삭제)를 보호한다.

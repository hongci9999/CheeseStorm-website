# 티어리스트 로직 개발 문서

## 1. 구조 개요

티어리스트는 두 탭으로 구성된다.

| 탭       | 역할                             | 신뢰도                         |
| -------- | -------------------------------- | ------------------------------ |
| 자동     | 전적 + 스탯 기반 참고 지표       | 낮음 (밸런싱된 팀 구조상 한계) |
| 큐레이션 | 운영자/권한 스트리머가 직접 배정 | 높음 (실제 팀 편성 기준)       |

> **왜 자동 티어가 참고용인가**: 내전은 실력 평균이 비슷하도록 팀을 의도적으로 구성한다.
> 따라서 승률만으로 개인 실력을 측정하면 전프로도 C티어에 배정될 수 있다.

## 2. 스트리머 자동 티어

### 2-1. 티어 기준

| 티어 | 혼합 점수 | 최소 경기  |
| ---- | --------- | ---------- |
| S    | 60%+      | 5경기      |
| A    | 55~60%    | 5경기      |
| B    | 45~55%    | 5경기      |
| C    | 35~45%    | 5경기      |
| D    | ~35%      | 5경기      |
| ?    | —         | 5경기 미만 |

### 2-2. 혼합 점수 공식

스탯 점수를 바로 혼합하면 스케일 충돌이 생기므로 **가상 승률로 변환** 후 혼합.

```
statWinRate = 0.2 + statScore × 0.6   // 0~1 점수 → 0.2~0.8 범위로 압축
finalScore  = α × winRate + (1-α) × statWinRate
```

- 스탯 최고(1.0) → statWinRate 0.80
- 스탯 최저(0.0) → statWinRate 0.20
- 스탯 중앙(0.5) → statWinRate 0.50

스탯이 티어를 과도하게 뒤집지 않도록 ±30% 범위로 제한.

스탯 커버리지(스탯이 기록된 경기 비율)에 따라 α 동적 조정:

| 스탯 커버리지 | α (winRate 가중치) | 설명                        |
| ------------- | ------------------ | --------------------------- |
| 30% 미만      | 0.80               | 스탯 신뢰도 낮음, 승률 위주 |
| 30~70%        | 0.50               | 동등 혼합                   |
| 70% 이상      | 0.35               | 스탯 충분, 비중 높임        |

### 2-3. 최근 폼

스트리머 자동 티어는 **시즌 전체 데이터**를 사용한다. 별도 최근 폼 가중치 없음.

이유:

- 시즌은 짧고 표본이 적음 (MIN_SAMPLE=5) — 최근 N경기로 더 자르면 신뢰도 하락
- 시즌 자체가 최근 폼 단위임 (`matches.season` 필드로 시즌 격리)
- 시즌 1 현재: 전체 데이터 = 최근 폼

최근 폼 상세는 **개인 프로필 페이지**에서 제공 (연승/연패, 최근 경기 결과).

### 2-4. 경기별 스탯 점수

스트리머의 **파생 역할이 아닌** 해당 경기에 플레이한 영웅의 역할을 기준으로 가중치 적용.

```
matchStatScore = Σ (statWeight[role][stat] × stat/min)
```

- 딜·힐·공성딜은 **분당(per-minute)** 값으로 정규화
- `dur`(경기시간) 없는 경기는 raw 값 사용 (fallback)
- KDA만 분당 아닌 비율값 (kills+assists / max(1, deaths))

### 2-5. 역할별 가중치 테이블

|            | KDA  |  xp/min  | heroDmg/min | siegeDmg/min | healing/min | selfHeal/min |
| ---------- | :--: | :------: | :---------: | :----------: | :---------: | :----------: |
| **암살자** | 0.25 |   0.20   |  **0.40**   |     0.05     |    0.00     |     0.10     |
| **탱커**   | 0.25 |   0.20   |  **0.20**   |     0.05     |    0.05     |   **0.25**   |
| **지원가** | 0.20 |   0.20   |    0.10     |     0.05     |  **0.40**   |     0.05     |
| **투사**   | 0.20 | **0.20** |    0.25     |   **0.10**   |    0.05     |     0.20     |
| **전문가** | 0.20 | **0.25** |    0.20     |   **0.20**   |    0.10     |     0.05     |

> - 탱커: heroDmg 반영 (이니시에이터 딜 기여 포함)
> - 투사: xp + siegeDmg 반영 (레오릭, 소냐 등 라인 압박 역할)
> - 전문가: 복합 지표 (아바투르, 메디브 등 다양한 플레이 스타일)
> - 지원가: heroDmg 소량 반영 (티란데 등 공격형 지원가 존재)

### 2-6. 정규화

같은 역할 그룹 내 플레이어끼리 min-max 정규화 (0~1):

```
normalized = (score - min) / (max - min)
```

- 역할 내 플레이어 2명 미만(min=max) → 정규화 불가, 모든 스탯을 중립값 0.5로 처리 (점수 0.5)
- 알 수 없는 영웅 역할 → 스탯 누적에서 제외 (0점 부여가 아니라 집계 제외 → 커버리지 하락)

## 3. 영웅 티어 (`hero-tier.ts`)

### 3-1. 집중도 반영 베이지안 승률

표본이 적거나 소수가 독점한 영웅의 극단값 억제:

```
top1Share = 최다 사용 스트리머 판수 / 전체 판수
C         = 3 + 5 × top1Share        (사전 강도)
score     = (wins + C/2) / (games + C)
```

사전 확률 50%. 독점 영웅(top1Share→1)일수록 사전 강도가 커져
승률이 50% 쪽으로 강하게 수축한다 — 좋은 쪽·나쁜 쪽 모두 과장 억제.
여러 명이 고루 쓰는 영웅은 판수가 많아도 추가 수축 없음.

> 구 방식(다양성 페널티: `distinctStreamers / games` 기반 점수 차감)은
> 판수가 많을수록 무조건 감점되고, 저승률 독점 영웅을 더 깎는 방향 오류가 있어 교체.

### 3-2. 최근 폼

**시즌 내 후반부 경기**의 승률 (`recentWinRate`) — 시즌 전체 경기 중 마지막 10경기.
3판 미만이면 `undefined` (표시 안 함).

> **시즌 1 특이사항**: 현재 모든 데이터가 첫 시즌이므로 전체 = 최근 폼.
> 시즌 초반엔 recentWinRate와 전체 승률의 차이가 거의 없음.
> 시즌이 쌓일수록 의미 있는 지표가 됨.

### 3-3. 정렬

티어 내 동률 시: 베이지안 승률 → 판수 → 영웅명 가나다순.

## 4. 구현 파일 구조

```
src/lib/
├── sample.ts        # MIN_SAMPLE = 5 (티어 최소 경기 수)
├── tier.ts          # calcTier, calcPlayerStats, groupStatsByTier
├── hero-tier.ts     # calcHeroTiers, groupHeroesByTier (베이지안)
├── stat-score.ts    # 역할별 가중치 + 경기별 스탯 점수 (구현 완료)
└── types.ts         # PlayerStats (recentWinRate, streak, topHero, statCoverage)
```

### `stat-score.ts` 실제 API

```typescript
// 전체 스트리머의 스탯 점수(0~1) + 커버리지(스탯 기록 경기 비율) 반환
export function calcAllStatScores(
  matches: Match[],
  streamerIds: string[],
): Map<string, { score: number; coverage: number }>;

// coverage → α (winRate 가중치). 스탯 신뢰도 낮을수록 승률 비중 높임.
export function statAlpha(coverage: number): number;

// stat score(0~1) → 가상 승률(0.2~0.8)
export function statToWinRate(statScore: number): number;
```

## 5. 시즌 개념

내전은 특정 기간에 집중되며 연 1~3회 시즌이 생길 수 있다.
티어리스트는 **시즌 단위**로 집계해야 과거 데이터가 현재 폼을 오염시키지 않는다.

### 현재 (시즌 1)

전체 데이터 = 시즌 1 데이터. 필터링 불필요.

### 향후 대응

`matches` 컬렉션에 `season` 필드 추가 권장:

```
matches: { ..., season: '2025-s1' }
```

- 티어 계산 시 특정 시즌만 필터링
- 큐레이션 티어도 시즌별 관리
- 날짜 범위 필터는 시즌 경계 관리가 어려우므로 비권장

## 6. 파라미터 튜닝 가이드

수동으로 조정할 수 있는 파라미터와 위치.

### `src/lib/stat-score.ts`

#### 역할별 스탯 가중치 (`ROLE_STAT_WEIGHTS`)

각 역할의 6개 가중치 합계는 **반드시 1.0** 이어야 함.

```ts
'암살자': { kda: 0.25, xpPerMin: 0.20, heroDmgPerMin: 0.40, siegeDmgPerMin: 0.05, healingPerMin: 0.00, selfHealPerMin: 0.10 }
```

- `heroDmgPerMin` ↑ → 딜러 고득점 유리
- `healingPerMin` ↑ → 힐러 고득점 유리
- `kda` ↑ → 죽지 않는 플레이 가중

#### statWR 범위 (`statToWinRate`)

```ts
return 0.2 + statScore * 0.6; // 스탯 최저→0.2, 최고→0.8
```

- 범위 넓힐수록 스탯이 티어를 더 많이 뒤집을 수 있음
- `0.3 + score * 0.4` → 0.3~0.7 (보수적)
- `0.2 + score * 0.6` → 0.2~0.8 (현재, 공격적)
- `0.0 + score * 1.0` → 스탯만으로 결정 (극단적)

#### α 가중치 (`statAlpha`)

```ts
if (coverage < 0.3) return 0.8; // 스탯 부족 → 승률 위주
if (coverage < 0.7) return 0.5; // 중간 → 동등
return 0.35; // 스탯 충분 → 스탯 비중 높임
```

- α 낮출수록 스탯 영향력 증가
- α=0.5 고정 → 승률/스탯 항상 동등

### `src/lib/tier.ts`

#### 티어 기준선 (`TIER_THRESHOLDS`)

```ts
{ tier: 'S', min: 0.60 },
{ tier: 'A', min: 0.55 },
{ tier: 'B', min: 0.45 },
{ tier: 'C', min: 0.35 },
{ tier: 'D', min: 0    },
```

- S가 비어있으면 기준선(현재 0.60)을 더 낮추는 방향 검토
- 전체적으로 너무 몰리면 기준선 간격 조정
